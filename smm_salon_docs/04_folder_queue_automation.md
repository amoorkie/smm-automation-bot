# Folder Queue Automation

## Назначение

Этот сценарий нужен для локальной автоматизации фото-очереди без ручного `/work` в Telegram.

Источник:
- `В обработку`

Промежуточное состояние:
- `В работе`

Результат:
- `Новое`
- `Обработано`

## Как работает MVP

Скрипт `scripts/process-folder-queue.mjs`:
- берёт самый старый поддерживаемый файл из `В обработку`;
- переносит его в `В работе`;
- прогоняет через тот же image+caption pipeline, что и основной бот;
- сохраняет готовый результат в `Новое`;
- переносит исходник в `Обработано`;
- отправляет фото и текст в Telegram по `OWNER_CHAT_ID` или `--chat-id`.

## Тестовый режим

Режим `--mode test` делает три вещи:
- переименовывает исходник и результат с префиксом `TEST__`;
- отправляет результат в Telegram как тестовый;
- после задержки rollback удаляет тестовый файл из `Новое` и возвращает исходник обратно в `В обработку`.

Параметр rollback:
- `--rollback-delay-sec 120`

Это окно в секундах, в течение которого можно увидеть движение файлов по папкам до отката.

## Поддерживаемые аргументы

```bash
node ./scripts/process-folder-queue.mjs \
  --root "C:\Users\amoor\Desktop\PHOTO\Анита" \
  --mode test \
  --background keep \
  --rollback-delay-sec 120
```

Аргументы:
- `--root` обязательный путь к корневой папке фото.
- `--chat-id` необязательный chat id для отправки результата. Если не указан, берётся `OWNER_CHAT_ID`.
- `--mode` `test` или `live`.
- `--background` `keep`, `neutral` или `blur`.
- `--rollback-delay-sec` задержка перед rollback в test-режиме.
- `--marker` текстовый префикс, по умолчанию `TEST`.

## Task Scheduler

Для разового теста через 5 минут:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\register-folder-queue-test-task.ps1 -RootDir "C:\Users\amoor\Desktop\PHOTO\Анита" -DelayMinutes 5
```

Скрипт регистрации создаёт одноразовый Windows Scheduled Task и запускает `process-folder-queue.mjs` через `node`.

## Категории

MVP поддерживает два варианта:
- файлы лежат прямо в `В обработку`;
- файлы лежат в подпапках категорий внутри `В обработку`.

Если категория содержит `бров`, материал идёт как `brows`. Все остальные категории обрабатываются как `hair`.

## Ограничения MVP

- Сейчас runner берёт только один файл за запуск.
- MVP рассчитан на одиночное фото.
- Категории пока нужны только для маршрутизации по папкам, не для отдельной prompt-логики, кроме `брови`.
- Для production-режима потом стоит добавить state log, антидубли и лимиты по количеству работ за прогон.
